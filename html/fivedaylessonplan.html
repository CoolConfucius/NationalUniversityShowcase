Welcome to the Web 

Welcome, let's talk about the web, the thing that you're using right now. The web is this fast network of connected computers.

What do we use websites for? Early on, researchers used websites to share their research. But now everybody uses it for pretty much everything. Maybe you use a social network to chat with your family, a photo website for sharing photos, or a news website to figure out what's going on in the world.

So what's your favorite website? Actually, I have a better question. What website do you wish existed, but doesn't yet? Maybe a website about yourself or about something that you love? The cool thing is you can program a website yourself. Start brainstorming what you want to make because you'll be able to do it soon enough. First, let's back up a bit and clear up some terminology. What is the "web," exactly? Well, like I said, it's a bunch of connected computers that have websites. When a computer is connected to the web and spitting out a website, we call that computer a "server" because it's "serving" the website. A website is written using three languages: HTML, for marking up the website content; CSS, for styling it; and JavaScript, for making it interactive. How do users like you actually see those websites? You use an app dedicated for website browsing, like Chrome, Firefox, or Internet Explorer, and we call those apps "browsers." Which browser do you regularly use? You must be using that browser from some sort of device, like a laptop or phone. We call those devices "clients."

There's lots more to learn about the web, but now I think it's about time to start having you write some HTML and CSS to make your own web pages.

 

Writing HTML

What is HTML?  

            Hyper Text Markup Language is a markup language, not a programming language. This means it describes content and structure. It focuses on semantics (meaning) to add context to what the coder is describing.

Text Editor for Coding

Before I go into the syntax of HTML, this is a good point for me to make sure all my students have a text editor used for coding. They will be using the text editor throughout the course. They are welcome to use either Sublime Text or Atom. I personally use Sublime Text, so it is the text editor I recommend. It is available in both Mac and Windows.

I will go around the classroom making sure each student has the text editor downloaded and opened on their computer. This process should be simple. If there are students who are stuck, I can help them quickly. I encourage students who are ready to help those who may be stuck.

HTML Tag Syntax

Pseudo Example: <tagname>Your content goes here. </tagname>

  • The tagname is a keyword surrounded by angle brackets < >
  • Come in sets. Opening tag and closing tag
  • Opening tag: <tagname>
  • Closing tag: </tagname> <= note the /
  • The element content is everything between the tags
  • Should all be lowercase

For example, the Paragraph <p> tag

<p>This is a paragraph.</p>

<p>This is a second paragraph.</p>

 

The two lines above produces the following:

This is a paragraph

This is a second paragraph

 

 

HTML is made to be nested

<p>This is a paragraph.</p>

<p>This is a <strong>second</strong> paragraph.</p>

 

The two lines above produces the following:

This is a paragraph

This is a second paragraph

 

In this example, the inline tag <strong> made the text “second” bold.

 

Now, I want to see them write html code using paragraph tags and a strong tag. They can either follow my example or write their own text. The paragraph can be a description of the webpage they are building throughout this period. Then, I want to see them open their html files in the browser. They can do this by dragging their file icon into a new tab in the browser.

Block Elements

  • Most HTML elements are block level
  • This means that they break the flow of the content
  • Define flow of the document
  • Some examples are: <h1>, <p>, <ul>, <table>, <div>

Unordered lists

 <ul>

 <li>Item 1</li>

 <li>Item 2</li>

 </ul>

 

Produces:

 

  • Item 1
  • Item 2

 

Ordered lists

 <ol>

 <li>Item 1</li>

 <li>Item 2</li>

 </ol>

 

Produces:

 

  1. Item 1
  2. Item 2

 

Unordered and ordered lists can be nested

 <ul>

            <li>

            Item 1

 <ol>

 <li>Sub item 1</li>

 <li>Sub item 2</li>

 </ol>

 </li>

 <li>Item 2</li>

 </ul>

 

Produces:

 

  • Item 1
  1. Sub item 1
  2. Sub item 2
  • Item 2

 

Now, I want to see my students creating lists with list items in their webpage. Their list can be instructions on baking a cake or anything that relates to their interests. I’ll also ask them to use a header tag to title their ongoing webpage.

Inline Elements

  • Inline elements do not break the flow of content
  • No newline is created
  • Most often used for styling
  • Some examples are <b>,

This is a <b>bold</b> statement.

<div> and <span> elements

  • <div> elements are block level containers that groups other elements
  • <span> elements are inline level containers that groups text

A span of te<span style=“font-weight:bold;”>xt is inl</span>ine.

Produces:

A span of text is inline.

 

There are a lot of tags

  • Headings are described with the <h1> to <h6> tags
  • Italics text is described with the <i> tags
  • Bold text is described with the <b> or <strong> tags
  • And so, so many more…

HTML Element Attributes

  • Attributes provide addi1onal informa1on of the elements
  • Attributes are specified in the start tag
  • Attributes are key="value" pairs:

This is a <span style="color: red;"></span>.

Some common attributes

  • id: For adding a unique id for an element
  • class: For adding one or more class names for an element for CSS targeting
  • style: For inline CSS style for an element
  • title: Extra title as a "tool tip"

These terms may be unfamiliar now, but they will make sense later on.

            Hyperlinks using the <a> tag

<a href="{url}" title="{text}">Link Text</a>

  • href attribute defines the URL that the link takes you to:

<a href="https://www.youtube.com/watch?v=6K16d6j9GRk">Gilbert Rocks!</a>

This is an example of an absolute path. Relative paths are more complex at this point and we don’t need them for now.

            <img> tag

<img src="batman-shark-battle.gif" alt="POW! BAM! BOOM! WHAM!"/>

  • <img> tag doesn't need a closing tag. It is self-closing using />
  • src attribute points to the URL of the image
  • alt attribute specifies an alternative text description of the image

A HTML document contains nested elements

<!DOCTYPE html>

<html lang="en">

 <head>

 <title>

 My Document's Title

 </title>

 </head>

 <body>

 My content goes here.

 </body>

</html>

 

It’s okay to not know completely what this means. Just know that this is the standard when writing an HTML file.

<html>, <head>, and <body>

  • The <html> tag wraps the whole document.
  • The <head> tag is where you define top level info such as the document title.
  • The <body> tag where your content belongs

Defining CSS in the <head>

CSS can be defined in the head tag like this:

<style>

 .fire {

 color: red;

 }

</style>

  • But the standard is to use an external CSS Stylesheet like this:

<link rel="stylesheet" href="styles.css"/>

This may seem confusing now but will make more sense later on when we go into CSS.

Defining JavaScript in the <head>

  • Example:

<script>

window.onload = function() {

 // Things that Batman does.

 alert( ‘POW! BAM! BOOM! WHAM!’ )

};

</script>

  • But the standard is to use external JavaScript like this:

<script src="app.js"></script>

This may seem confusing now but will make more sense later on when we go into JavaScript.

            Break

If the period is not over at this point, I think my students deserve a little break for learning so much material in one period. If there is more time, then I would like to introduce them to more advanced html tags such as the input tag, button tag, or table tag.

With an input tag, students can create an input box for users to enter values such as a name or phone number. They can also create buttons for users to click on or tables with information related to their webpage.

Until the period is over, I would like my students to make their webpage as cool as they can by using what we learned or beyond. This is also an opportunity for students to ask me questions.

If there is a lot of time left, then I may go into the next topic: CSS.

Writing CSS

 

Teacher: Benjamin Huang

Subject: Computer Science / Programming: Developing Web Applications

Lesson Title: Writing CSS Code

Classroom Details

Time: 20-30 minutes

Equipment: Computers, Projector

Resources:  Sublime Text Editor

Content Details

Grade: 6-12

Standard(s): 9-12.DA.8

Translate between different representations of data abstractions of real-world phenomena, such as characters, numbers, and images.

Objective: Students should be able to, through defining CSS classes and ids, transform their web page in ways including: font family, font size, text color, background color, height, width, etc.
 

Time in mins.

Lesson Element

What is the teacher doing?

What are the students doing?

0-1

mins.

Student Friendly Objective

Tell students:

By the end of the lesson you should be able to…

Write CSS code in to style the static web page they have built from writing HTML code.

Students are getting excited imagining how they can make their web page look more exciting.

  

0-1

Mins

Link to prior knowledge

(pre-assessment)

I remind my students they have learn how to build static web pages by writing HTML code. I will tell them that they will now learn how to style their web page by writing CSS code.

In the pre-assessment, I interview my students or send them forms to determine their knowledge on CSS.

The students are reflecting on what they have learned previously.

In the pre-assessment, they were communicating to me their knowledge on CSS.

  

10-15

Mins

Input / Modeling

I will be briefly going over the common features of CSS. For each feature, I will demonstrate an example. I will be showing students how to: decorate text, change the font color, Font family, size, and color; Background colors; widths, heights, alignments; The Box Model: Borders, paddings, and margins. On writing CSS selectors, I will show them how to write descendent selectors, class selectors, and id selectors.

The students are paying attention. Ideally, the students should be writing and testing my examples on their computers. This would be effective practice for them.

  

5-10 mins

Guided Practice

(checking for understanding)

For each topic on CSS, I will invite students to write CSS code with me. The projector will show what is on my text editor. I may ask the students questions. What should I include in the editor? Do you see a bug with this? What is my next step? Can I improve the code?

The students are answering my questions and asking their questions if they have any. They are following along the guided practice.

  

5-10 mins

Independent Practice

(ongoing assessment)

This is a good opportunity for students to take the next step on their ongoing project, the web application. I will ask my students to bring up the static web page they have built in HTML and style it by writing CSS code. I will be available to help students who need it.

The students are styling their webpage by writing CSS code. They are welcome to ask me questions if they are stuck.

  

5-30 mins

Closure

(post-assessment)

I will be asking my students to present their projects. I will evaluate the CSS code they have written and ask the students questions. What is your thought process when writing this line of code? Can you explain what this code does?

The students are presenting their project to me and answering my questions. They are communicating their learning clearly to me by showing me their code and their thought process.
